🌐 SportSwear

Se propone el desarrollo de una plataforma web de E-commerce para la venta de ropa, que cuente con un catálogo organizado por categorías, un sistema de búsqueda y filtrado, un módulo de registro e inicio de sesión, un carrito de compras y un sistema de pago en línea.

🎯 1. Objetivo General

Diseñar y desarrollar un sistema eCommerce para la venta de ropa que permita a los clientes visualizar un catálogo de productos, realizar compras en línea de forma segura y 
gestionar sus pedidos, garantizando una experiencia de uso clara, intuitiva y adaptable a distintos dispositivos.


🌍 2. Contexto de Uso

El sistema estará dirigido a personas interesadas en adquirir prendas de vestir a través de internet, utilizando dispositivos móviles. Los clientes podrán acceder al sistema desde 
cualquier lugar con conexión a la red, consultar productos, registrarse, realizar compras y hacer seguimiento a sus pedidos. El sistema será utilizado tanto por clientes finales como por administradores encargados de gestionar los productos, precios y pedidos.


📋 3. Requerimientos del Sistema
3.1 Requerimientos Funcionales

RF1. El sistema debe permitir la visualización de un catálogo de productos organizado por categorías (por ejemplo: tipo de prenda, talla, público objetivo).
RF2. El sistema debe mostrar la información detallada de cada producto, incluyendo imágenes, precio, descripción y tallas disponibles.
RF3. El sistema debe permitir la búsqueda de productos y el filtrado por criterios como talla, color y rango de precio.
RF4. El sistema debe permitir el registro de nuevos usuarios y el inicio de sesión de usuarios existentes.
RF5. El sistema debe permitir a los usuarios gestionar su perfil personal y sus direcciones de envío.
RF6. El sistema debe permitir a los usuarios agregar, modificar o eliminar productos del carrito de compras.
RF7. El sistema debe permitir la realización de compras mediante un proceso de pago en línea.
RF8 El sistema debe guardar la información de registro del usuario, para que cuando entre a la página a realizar la compra la información la traiga predeterminadamente y que el sistema no le pida datos registrados, solo la dirección de envío.
RF9. El sistema debe permitir la generación y confirmación de pedidos una vez realizado el pago.
RF10. El sistema debe permitir a los usuarios realizar el seguimiento del estado de sus pedidos.
RF11. El sistema debe permitir la visualización del historial de compras del usuario.
RF12. Que el sistema permita cerrar sesión 


3.2 Requerimientos No Funcionales

RNF1. El sistema debe contar con un diseño adaptativo que permita su correcta visualización y uso en dispositivos móviles, tablets y computadores.
RNF2. El sistema debe garantizar la seguridad de la información de los usuarios, especialmente los datos personales y de pago.
RNF3. El sistema debe permitir tiempos de respuesta adecuados en la carga del catálogo y las páginas de productos.
RNF4. El sistema debe ser accesible mediante un navegador web sin necesidad de instalaciones adicionales.
RNF5. El sistema debe contar con un diseño de interfaz intuitivo y fácil de usar para cualquier tipo de usuario.
RNF6. El sistema debe permitir la integración con una pasarela de pagos en línea .
RNF7. El sistema debe utilizar una base de datos para el almacenamiento de información de usuarios, productos y pedidos.
RNF8. El sistema debe ser escalable para permitir el aumento de usuarios y productos sin afectar su funcionamiento.
RNF9. El sistema debe contar con mecanismos de respaldo de la información (copias de seguridad).
RNF10. El sistema debe cumplir con buenas prácticas de desarrollo web y estándares básicos de accesibilidad.
